# Руководство пользователя Auto Translation Framework

[English](../user-guide.html) | **Русский** | [简体中文](../zh-cn/user-guide.html)

Это руководство описывает пользовательский процесс Auto Translation Framework 1.0.11 для RimWorld 1.6.

## Требования

- RimWorld 1.6.
- [Harmony](https://steamcommunity.com/sharedfiles/filedetails/?id=2009463077), загруженный перед Auto Translation Framework.
- Доступ в интернет для web/API-переводчиков и опциональной общей библиотеки.
- API-ключ нужен только в том случае, если его требует выбранный провайдер.

## Установка

Рекомендуемый способ установки — [Steam Workshop](https://steamcommunity.com/sharedfiles/filedetails/?id=3759370650).

1. Подпишитесь на Harmony и Auto Translation Framework.
2. Откройте меню **Моды** в RimWorld.
3. Включите Harmony перед Auto Translation Framework.
4. После изменения списка активных модов перезапустите RimWorld.

Существующие человеческие переводы по умолчанию сохраняются и имеют приоритет над сгенерированным результатом.

## Первый запуск

1. Выберите предпочтительный язык интерфейса RimWorld.
2. Откройте **Настройки > Настройки модов > Auto Translation Framework**.
3. На вкладке **Главное** включите переводчик и оставьте выбранным пункт **Использовать язык интерфейса RimWorld**.
4. На вкладке **Переводчики** используйте **Google Translate Web** для стандартной конфигурации без ключа.
5. Закройте окно настроек и дождитесь сканирования активных модов.
6. Вернитесь на вкладку **Моды**, чтобы увидеть покрытие, ожидающие строки и готовые переводы.

Стандартный провайдер работает без API-ключа. Платные API, локальные модели и общая библиотека являются дополнительными возможностями.

## Вкладки настроек

| Вкладка | Назначение |
| --- | --- |
| **Главное** | Главный переключатель, целевой язык и компактный статус конвейера. |
| **Переводчики** | Выбор провайдера, учётные данные, проверка здоровья, профили, пакеты и multi-provider режим. |
| **Моды** | Охват по модам, результаты сканирования, прогресс, повтор, применение, сброс и поиск. |
| **Редактор** | Открывает полноэкранное окно ручной правки переводов. |
| **Библиотека** | Общая база, авторские пакеты, каталог, загрузка, оценки и опциональная отправка. |
| **Расширенные** | Области перевода, производительность, фоновая очередь, проверка, сеть и локальное хранилище. |
| **Отладка** | Состояние конвейера, диагностические журналы, восстановление и сброс настроек. |

## Области перевода

### Keyed

Keyed — стандартные ключи локализации RimWorld, используемые меню, сообщениями, предупреждениями и системами модов. Область включена по умолчанию.

### DefInjected

DefInjected охватывает поддерживаемые поля Def: названия, описания, черты, мысли, предметы, здания, исследования, события, hediff и другие подобные данные. Область включена по умолчанию.

### Захардкоженный C# UI

Некоторые моды выводят английский текст напрямую из C#, не используя языковые файлы RimWorld. Runtime-слой отслеживает поддерживаемые пути интерфейса и переводит известный текст во время отрисовки. Поддерживаются некоторые надписи, кнопки, контекстные меню, команды, подсказки, строки осмотра, письма и сообщения.

Покрытие runtime-строк работает по принципу best effort. Если в очень большой сборке снижается производительность, отключите только **Переводить захардкоженные C# UI-строки** — Keyed и DefInjected можно оставить включёнными.

### Файлы Strings

Общий перевод `Strings` пока отключён. Эти файлы могут содержать чувствительные к формату правила и требуют отдельной безопасной обработки.

## Провайдеры перевода

### Провайдеры без ключа

Встроенные web-варианты включают Google Translate Web, Yandex Web, Bing Translator Web и MyMemory Web. Google Translate Web используется как стандартный одиночный провайдер.

Публичные web-сервисы могут ограничивать частоту запросов, менять поведение или временно становиться недоступными. Проверка здоровья, задержка повторов и сетевой предохранитель не дают фоновой очереди бесконечно повторять ошибки.

### API и локальные провайдеры

Меню также поддерживает:

- ChatGPT / OpenAI;
- Claude;
- локальные и удалённые OpenAI-совместимые серверы;
- OpenRouter;
- DeepSeek;
- Gemini;
- Google Cloud Translation;
- Microsoft Translator;
- DeepL и DeepL Pro;
- LibreTranslate;
- Apertium и совместимые прокси перевода.

Для API-провайдера выберите его и заполните адрес, модель, API-ключ и дополнительные поля, показанные интерфейсом. Перед большим объёмом работы используйте кнопку **Проверить провайдеры сейчас**.

Учётные данные сохраняются вместе с настройками модов RimWorld. Не публикуйте и не передавайте файл настроек, содержащий приватные ключи.

### Режим нескольких провайдеров

Multi-provider режим распределяет ожидающие строки между несколькими настроенными сервисами и может выполнять запросы параллельно. Для каждого слота задаётся собственный параллелизм. Начинайте с умеренных значений: повышение числа запросов способно вызвать ограничение со стороны сервиса без улучшения качества.

Профили позволяют хранить несколько конфигураций одного провайдера, например локальную модель и удалённый OpenAI-совместимый сервис.

## Управление отдельными модами

Вкладка **Моды** показывает активную сборку, количество найденных, ожидающих и переведённых строк.

- Отключите перевод для мода, у которого уже есть предпочтительный языковой пакет или который не нуждается в автопереводе.
- Кнопка **Применить** запускает применение готовых записей.
- Кнопка **Довести перевод** возвращает отсутствующие или неудачные записи в рабочий процесс.
- Кнопка **Сбросить** перестраивает сгенерированное состояние выбранного мода.
- Используйте **Пересканировать активные моды сейчас** после изменения области перевода или списка модов.

Отключённые моды остаются в статистике сканирования, но их новые строки не добавляются в очередь.

## Редактор переводов

Откройте **Редактор > Открыть редактор переводов**, чтобы запустить полноэкранное рабочее окно.

Редактор позволяет:

- группировать память по модам;
- искать оригиналы, ключи и готовые переводы;
- вручную редактировать переведённый текст;
- возвращать запись в автоматический перевод;
- восстанавливать или копировать оригинал;
- сохранять человеческие исправления в локальную память;
- отправлять выбранный мод как авторский перевод, если настроен обмен с библиотекой.

После ручного сохранения запускаются планировщики runtime-применения и генерации пакета. Часть изменений появится во время игры, но чувствительные к формату Def могут полностью обновиться только после следующего запуска RimWorld.

## Общая библиотека переводов

Библиотека является опциональной и не используется, пока её элементы управления не включены.

Доступны следующие процессы:

- загрузка повторно используемой проверенной базы;
- просмотр и загрузка авторских пакетов;
- оценка выбранного авторского пакета;
- отправка полного авторского перевода одного мода;
- отправка человеческих исправлений, если соответствующая настройка включена.

Загрузка и отправка управляются раздельно. Включение загрузки не включает отправку. Для операций отправки может потребоваться авторизация сервиса, а материалы могут проходить проверку перед публикацией в доверенной базе.

Библиотека уменьшает число повторных запросов: если совпадающий перевод уже существует, он используется вместо повторной обработки того же текста.

## Локальные файлы и безопасность

Auto Translation Framework не изменяет исходные файлы других модов из Steam Workshop.

Локальное хранилище содержит:

- `TranslationMemoryV2.jsonl` — постоянную память переводов;
- `GeneratedPacks` — сгенерированные данные локализации;
- `LibraryModStateV1.jsonl` — локальное состояние общей библиотеки;
- `HumanCorrectionsV1.jsonl` — очередь ручных исправлений;
- `PipelineDiagnostics.log` — диагностические события.

Используйте кнопку **Открыть папку локальных файлов** в расширенных или отладочных настройках, чтобы открыть правильную папку для текущей платформы.

Сгенерированный XML является кэшем фреймворка. Постоянные исправления вносите через редактор переводов, а не напрямую в сгенерированные файлы.

## Решение проблем

### Ничего не переводится

1. Убедитесь, что на вкладке **Главное** включён основной переключатель.
2. Проверьте определённый целевой язык.
3. Убедитесь, что нужный мод включён на вкладке **Моды**.
4. Проверьте области Keyed, DefInjected и hardcoded UI в расширенных настройках.
5. Проверьте состояние сервиса на вкладке **Переводчики**.
6. Используйте **Пересканировать активные моды сейчас** или **Перезапустить workflow перевода**.

### Провайдер постоянно выдаёт ошибку

- Запустите **Проверить провайдеры сейчас**.
- Проверьте адрес, модель, API-ключ, регион и дополнительные поля сервиса.
- Уменьшите параллелизм и размер пакета при ограничениях частоты или тайм-аутах.
- Дождитесь указанного времени повтора, не перезапуская запросы вручную.
- Выберите другой провайдер, чтобы исключить временную проблему конкретного сервиса.

### Часть текста остаётся на английском

- Проверьте, включён ли перевод захардкоженного C# UI.
- Один раз откройте соответствующее окно мода, чтобы runtime-слой увидел и поставил строку в очередь.
- Проверьте, не является ли текст динамическим, графическим или созданным неподдерживаемым пользовательским UI.
- После большого пакета DefInjected перезапустите RimWorld, чтобы XML-кэш загрузился с начала запуска.

### Снижается производительность большой сборки

- Сначала отключите захардкоженный C# UI и сравните результат.
- Уменьшите число потоков сканера и параллельных запросов.
- Не включайте отладочное логирование во время обычной игры.

### Нужны дополнительные данные для диагностики

Откройте вкладку **Отладка**, изучите состояние конвейера и используйте **Открыть лог диагностики пайплайна**. При сообщении о проблеме на [странице Steam Workshop](https://steamcommunity.com/sharedfiles/filedetails/?id=3759370650) приложите соответствующий статус и короткое описание воспроизведения. Не публикуйте API-ключи и приватные токены.

## Текущие ограничения

- Runtime-слой не может перехватить каждый пользовательский способ отрисовки или динамически созданный текст.
- Провайдер способен выбрать неправильную терминологию даже после успешной проверки форматирования.
- Общие файлы `Strings` пока не переводятся.
- Некоторые готовые переводы появляются везде только после перезапуска.
- Доступность провайдеров и библиотеки зависит от внешних сетевых сервисов.
